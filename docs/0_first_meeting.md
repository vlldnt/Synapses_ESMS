# SYNAPSES ESMS — Project Summary 

## Overview *(03/03/2026)*
**SYNAPSES ESMS** is a social-impact project aiming to develop an **AI-assisted web application** for professionals working in the **social and medico-social sector (ESMS)**.

Professionals in this sector spend a significant amount of time producing administrative and professional documents such as intervention reports, educational assessments, personalized support plans, meeting summaries, and activity reports.  
The goal of SYNAPSES ESMS is to **reduce this administrative burden** by providing an **AI-assisted writing tool** that helps structure and generate professional documents.

The tool is designed to **support professionals rather than replace them**, allowing users to input key information about a situation and receive a structured draft that can then be reviewed and edited.

A major requirement of the project is to ensure **data privacy and confidentiality**, particularly by guaranteeing that **no personal or identifiable data is transmitted to the AI system**.

---

## Project Goal
The objective is to deliver a **functional MVP (Minimum Viable Product)** within **3 months (March–May 2026)**.

This prototype must:
- demonstrate the practical value of AI-assisted writing in ESMS contexts
- provide several usable document generation modules
- guarantee **data anonymization before any AI request**
- implement **user roles and access control**
- be stable enough for a **complete demonstration**

The project also serves as a **learning and graduation project** for the student developers involved.

---

## Core Features (MVP)

The MVP focuses on **AI-assisted generation of professional documents**, including:

- **Intervention reports**
- **Personalized support plans (PPA)**
- **Educational notes and evaluation reports**
- **Meeting summaries**
- **Management reporting**
- **Activity and annual reports**
- **Internal and external evaluation documentation**
- **Support for responses to project calls**

An optional feature may include a **professional monitoring / knowledge module**.

### Performance targets
- Text generation time **under 10 seconds**
- **No identifiable data sent to the AI**
- A **stable version without blocking bugs**

## Technical Approach

The solution will be built as a **web application** accessible through a browser on both **desktop and mobile devices**.

Key technical principles include:

- Integration with an **AI API (Mistral)**
- **Simple and maintainable architecture** to allow rapid MVP development
- Clear separation between **user input, anonymization, and AI requests**
- Strong focus on **data protection and confidentiality**

The system should remain **lightweight and scalable**, allowing future improvements and additional modules.
